# 평가 기준표

Generator와 Evaluator가 공유하는 품질 기준.

---

## 채점 항목

### 1. 디자인 품질 (비중: 40%)
색상, 글꼴, 레이아웃이 하나의 분위기를 만드는가?

합격: 일관된 색상 팔레트, 글꼴 2~3종 통일, 여백 규칙 일정, "하나의 브랜드"로 느껴짐
불합격: 색상 산만, 섹션마다 다른 디자인, AI slop 패턴 (보라색 그라데이션 + 흰색 카드)

### 2. 독창성 (비중: 30%)
이 프로젝트만의 개성이 느껴지는가?

합격: 템플릿과 확연히 다른 점이 있음, 레이아웃/색상/인터랙션에서 예상 밖 선택, AI가 만든 건지 구분 어려움
불합격: Bootstrap 기본 느낌, 뻔한 히어로→기능→팀→CTA 구조, 기억에 남는 요소 없음

### 3. 기술적 완성도 (비중: 15%)
기본기가 갖춰져 있는가?

합격: 반응형 정상 작동, 글자/간격 일관, 색상 대비 충분, 에러 없음
불합격: 모바일 깨짐, 텍스트 읽기 어려움, 콘솔 에러

### 4. 기능성 (비중: 15%)
직관적으로 사용할 수 있는가?

합격: 네비게이션 명확, CTA 눈에 띔, 스크롤 자연스러움, 폼 정상 작동
불합격: 버튼 못 찾음, 링크 안 됨, 정보 순서 뒤죽박죽

---

## 판정

가중 점수 = (디자인×0.4) + (독창성×0.3) + (기술×0.15) + (기능×0.15)

- 7.0 이상 → 합격
- 5.0~6.9 → 조건부 합격 (피드백 반영 후 재검수)
- 5.0 미만 → 불합격
- 디자인 또는 독창성 4점 이하 → 무조건 불합격

---

## 피드백 형식

```
**전체 판정**: [합격 / 조건부 합격 / 불합격]
**가중 점수**: X.X / 10.0

**항목별 점수**:
- 디자인 품질: X/10 — [한 줄 코멘트]
- 독창성: X/10 — [한 줄 코멘트]
- 기술적 완성도: X/10 — [한 줄 코멘트]
- 기능성: X/10 — [한 줄 코멘트]

**구체적 개선 지시**:
1. [어디를 어떻게 고칠 것]
2. [어디를 어떻게 고칠 것]

**방향 판단**: [현재 방향 유지] 또는 [완전히 다른 접근 시도]
```
