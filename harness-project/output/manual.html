📋 시스템 구성

꿈-드림 페스티벌 미래기술존 운영 시스템은 웹 기반으로, 별도 앱 설치 없이 브라우저에서 모든 기능을 사용할 수 있습니다.

📊 운영 대시보드

관리자/참가자 듀얼 모드. 실시간 관제맵, 통계, 스탬프 보드, 공지 발송

📱 QR 체크인

참가자 등록, QR 코드 발급, 카메라 스캔 체크인, 체크인 현황 대시보드

🖥️ 키오스크 모드

부스 안내 전시용. 대형 화면/태블릿 최적화, 자동 스크린세이버

📜 인증서 발급

참가 확인서, 스탬프 완료 인증서 생성. 인쇄 및 PDF 다운로드

🎮 체험 프로그램 (15개)

AI 모션 분석, 객체인식, 창작 스튜디오 등 인터랙티브 체험 콘텐츠

🖥️ 백엔드 서버

Node.js 실시간 동기화. 다중 기기 데이터 공유, WebSocket 실시간 알림

⚙️ 설치 방법

1단계: Node.js 설치

서버 PC에 Node.js (v18 이상)가 설치되어 있어야 합니다.

# Node.js 버전 확인
node --version    # v18.0.0 이상

# 설치되지 않은 경우
# https://nodejs.org 에서 LTS 버전 다운로드

2단계: 서버 의존성 설치

# 프로젝트 디렉토리로 이동
cd harness-project/server

# 의존성 설치 (최초 1회)
npm install

3단계: 서버 시작

# 서버 실행
npm start

# 또는 직접 실행
node server.js
✅ 서버가 정상 시작되면 http://localhost:3000에서 전체 시스템에 접근할 수 있습니다.
같은 네트워크의 다른 기기에서는 http://서버IP:3000으로 접속합니다.

4단계: 서버 IP 확인

# macOS / Linux
ifconfig | grep "inet " | grep -v 127.0.0.1

# Windows
ipconfig | findstr IPv4

# 예: 192.168.1.100 → 다른 기기에서 http://192.168.1.100:3000

서버 없이 사용하기 (오프라인 모드)

# 단순 정적 파일 서버로도 동작 (동기화 없음)
cd harness-project/output
python3 -m http.server 8000

# http://localhost:8000 에서 접근
⚠️ 오프라인 모드 제한 서버 없이 사용하면 기기 간 데이터 동기화가 되지 않습니다. 각 기기에서 독립적으로 localStorage에 저장됩니다. 웹캠이 필요한 체험은 HTTPS 또는 localhost에서만 동작합니다.

🌐 네트워크 구성

권장 네트워크 구성

  1. WiFi 공유기를 운영 본부에 설치
  2. 서버 PC를 유선(LAN)으로 공유기에 연결 (안정성 확보)
  3. 나머지 기기는 WiFi로 연결
  4. SSID 예: DreamFest-Staff (운영진용), DreamFest-Guest (참가자용)
⚠️ 네트워크 주의사항
  • 행사장 WiFi와 외부 인터넷이 분리된 환경에서는 CDN 리소스(폰트, 라이브러리)가 로드되지 않을 수 있습니다.
  • 사전에 한 번 모든 페이지를 열어 Service Worker 캐시를 채워두면 오프라인에서도 동작합니다.
  • 웹캠 기반 체험 (프로그램 01~07, 10, 12)은 localhost 또는 HTTPS에서만 카메라 접근이 가능합니다.

HTTPS 없이 웹캠 사용하기

Chrome에서 비보안 origin의 카메라 접근을 허용하는 방법:

# Chrome 실행 시 플래그 추가 (개발/행사용)
# macOS:
open -a "Google Chrome" --args --unsafely-treat-insecure-origin-as-secure="http://192.168.1.100:3000"

# Windows:
chrome.exe --unsafely-treat-insecure-origin-as-secure="http://192.168.1.100:3000"

또는 Chrome 주소창에 chrome://flags/#unsafely-treat-insecure-origin-as-secure를 입력하여 서버 주소를 추가합니다.

✅ 사전 점검 체크리스트

D-1 (전날)

  • 서버 PC에 Node.js 설치 확인
  • npm install 완료 확인
  • npm start로 서버 정상 기동 확인
  • 모든 기기에서 http://서버IP:3000 접속 확인
  • 웹캠 사용 부스 PC에서 카메라 접근 권한 허용 확인
  • 각 존 키오스크 디스플레이 정상 표시 확인
  • QR 체크인 태블릿 카메라 스캔 테스트
  • 인증서 인쇄 프린터 연결 확인
  • 모든 페이지를 한 번씩 열어 Service Worker 캐시 채우기
  • WiFi 신호 강도 확인 (모든 부스 위치에서)

D-Day (당일 아침)

  • 서버 PC 전원 켜기 및 npm start 실행
  • 운영 대시보드 열기 (관리자 모드)
  • 체크인 태블릿에서 QR 체크인 페이지 열기
  • 각 부스 PC에서 해당 프로그램 페이지 열기
  • 키오스크 디스플레이 전원 및 페이지 열기
  • 대시보드 우하단 "Server Connected" 표시 확인
  • 테스트 체크인 1건 수행 후 대시보드 반영 확인

🎉 개장 절차

1 서버 시작 (운영본부)

서버 PC에서 cd server && npm start 실행. 콘솔에 "꿈-드림 페스티벌 미래기술존 서버" 배너 확인.

2 운영 대시보드 확인

http://서버IP:3000 접속 → 관리자 모드 선택 → 관제 맵에서 15개 부스 표시 확인.

3 체크인 태블릿 준비

입구 태블릿에서 http://서버IP:3000/qr-checkin.html 접속. "참가자 등록" 탭에서 테스트 등록 1건 수행.

4 키오스크 디스플레이 배치

각 존 입구에 키오스크 설치. http://서버IP:3000/kiosk.html?booth=N (N=부스번호). 전체화면 모드(F11) 활성화.

5 부스 체험 PC 준비

각 부스 PC에서 해당 프로그램 페이지 접속.
예: 1번 부스 → http://서버IP:3000/programs/01-ai-body-scanner.html

6 개장!

참가자 입장 → 입구에서 QR 등록 → 자유 부스 체험 → 스탬프 수집 → 인증서 발급

참가자 모드

상단 "참가자" 버튼을 클릭하면 참가자 전용 화면이 표시됩니다:

  • 스탬프 보드: 15개 부스 체험 완료 현황 (완료 부스 강조 표시)

  • QR 체크인: 내 QR 코드 표시 및 체크인 이력
  • 리더보드: 댄스/분류/프롬프트 챌린지 순위
  • 알림: 운영 공지 수신

📱 QR 체크인 운영

접속 URL

http://서버IP:3000/qr-checkin.html

운영 흐름

1. 참가자 등록 (입구)

  1. "참가자 등록" 탭 선택
  2. 닉네임 입력, 아바타 선택
  3. "등록" 버튼 클릭 → DF-XXXX ID 자동 생성
  4. 화면에 QR 코드 표시 → 참가자에게 보여주거나 스크린샷 안내

2. 부스 체크인

  1. "부스 체크인" 탭 선택
  2. 부스 선택 (드롭다운)
  3. 카메라 스캔: "카메라 스캔 시작" → 참가자 QR 비추기 → 자동 인식
  4. 수동 입력: "수동 입력" 탭 → DF-XXXX ID 입력 → "체크인" 클릭

3. 체크인 현황

  • "체크인 현황" 탭에서 전체 통계 및 타임라인 확인
  • 존별/부스별 체크인 수 차트
  • CSV 내보내기 기능
💡 팁 참가자가 QR 코드를 분실한 경우, "수동 입력"으로 ID를 직접 입력하거나 대시보드 참가자 목록에서 검색할 수 있습니다.

🔧 트러블슈팅

자주 발생하는 문제

"Server Connected"가 표시되지 않아요

  • 서버가 실행 중인지 확인: 서버 PC에서 콘솔 출력 확인
  • 같은 네트워크에 연결되어 있는지 확인
  • 서버 IP와 포트가 정확한지 확인: http://서버IP:3000/api/stats 접속 테스트
  • 방화벽에서 3000번 포트를 허용했는지 확인

웹캠이 작동하지 않아요

  • localhost 또는 HTTPS에서만 카메라 접근 가능
  • Chrome 주소창 왼쪽 자물쇠 아이콘 → 카메라 "허용"으로 변경
  • 다른 앱이 카메라를 사용 중이면 해제
  • Chrome 플래그로 비보안 origin 허용 (위 네트워크 구성 섹션 참조)

페이지가 로드되지 않아요 (오프라인)

  • 인터넷 연결이 없으면 CDN 리소스(폰트, 라이브러리)가 로드되지 않을 수 있음
  • 사전에 한 번 모든 페이지를 열어 Service Worker 캐시를 채워두면 해결
  • SW가 등록되었는지 확인: Chrome DevTools → Application → Service Workers

QR 스캔이 안 돼요

  • 카메라 권한 허용 확인
  • QR 코드에 충분한 조명 확보
  • 카메라와 QR 코드 사이 10~30cm 거리 유지
  • 수동 입력 모드로 대체 가능

데이터가 다른 기기에 동기화되지 않아요

  • 모든 기기에서 같은 서버 URL을 사용하는지 확인
  • 브라우저 페이지 우하단에 "Server Connected" 표시 확인
  • 서버를 재시작하면 기존 데이터는 server/data/에 보존됨

PDF 다운로드가 안 돼요

  • 팝업 차단 해제 확인
  • CDN(html2canvas, jsPDF)이 로드되었는지 확인 (인터넷 필요)
  • 대안: "인쇄" 기능 사용 → "PDF로 저장" 선택

서버 관련

서버가 갑자기 종료되었어요

# 서버 다시 시작
cd server && npm start

# 데이터는 server/data/ 에 안전하게 보존됩니다

데이터를 초기화하고 싶어요

# API로 초기화 (서버 실행 중)
curl -X POST http://localhost:3000/api/sync/reset \
  -H "Content-Type: application/json" \
  -d '{"confirm":"RESET_ALL_DATA"}'

# 또는 파일 직접 삭제
rm -rf server/data/
# 서버 재시작하면 빈 데이터로 시작

서버 포트를 변경하고 싶어요

# 환경변수로 포트 지정
PORT=8080 npm start

# 또는 server.js 상단의 PORT 변수 수정

🚨 비상 대응

🔴 서버 완전 장애 시
  1. 각 기기는 오프라인 모드(localStorage)로 자동 전환됨
  2. 체험 프로그램은 서버 없이 독립 동작
  3. 체크인은 "수동 입력" 또는 종이 명단으로 대체
  4. 서버 복구 후 각 기기에서 페이지 새로고침하면 자동 재연결
🔴 네트워크 전체 장애 시
  1. Service Worker 캐시로 페이지는 계속 로드 가능
  2. CDN 의존 리소스(폰트)는 캐시에서 제공됨
  3. 각 기기에서 독립적으로 체험 계속 진행
  4. 체크인 데이터는 각 기기 localStorage에 저장되며, 네트워크 복구 후 동기화 가능
⚠️ 특정 부스 PC 장애 시
  1. Chrome 새로고침(Ctrl+Shift+R) 시도
  2. 안 되면 Chrome 종료 후 재시작
  3. 그래도 안 되면 다른 노트북/태블릿으로 교체
  4. 대시보드에서 해당 부스 상태를 "점검중"으로 변경
