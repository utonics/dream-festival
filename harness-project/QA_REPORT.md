### 피드백 2: 혼잡도 맵을 실제 맵 형태로 교체

`renderCongestion()` 함수(line 2012~2098)가 SVG 기반 간소화 평면도로 완전히 재작성됨. 5개 존이 `zoneBlocks` 좌표로 공간적 배치되고, 존 내부에 부스 노드가 원형으로 표시됨. 혼잡 부스의 펄스 애니메이션(`<animate>` 태그), 존별 혼잡도 인디케이터(우상단 원), 부스 간 연결 패스(dash 라인)도 포함. SPEC이 요구한 "관리자 플로어맵의 참가자 버전"에 부합함. **이 항목은 개선됨.**

### 피드백 3: 간트 차트 현재 시간선 버그 수정
**판정: 반영됨**

이전에 문제가 된 CSS calc 표현식이 제거됨. `renderTimetable()` 함수(line 1412~1427)에서 JavaScript로 직접 픽셀 위치를 계산:
```js
const gridRect = ganttGrid.getBoundingClientRect();
const labelWidth = 180;
const timeAreaWidth = gridRect.width - labelWidth;
const nowPct = (nowHour - startHour)/hours;
const nowLeft = labelWidth + nowPct * timeAreaWidth;
nowLine.style.left = nowLeft + 'px';
```
BoundingClientRect 기반 실제 픽셀 계산으로 정확한 위치 지정. **이 항목은 개선됨.**

- **그러나 실시간 시뮬레이션(line 2507~2518)에서 여전히 `Math.random()`으로 `current`와 `queue` 값을 변경**하고, 이것이 혼잡도 맵과 플로어맵에 반영됨. 이것은 "실시간 시뮬레이션"의 의도적 설계이므로, 이전 피드백에서 지적한 "매 렌더링마다 수치가 달라지면 데이터를 신뢰할 수 없다"와는 성격이 다름 — 시뮬레이션은 데모 특성상 허용 가능. 통계 차트 렌더링의 핵심 문제는 해결됨. **대부분 개선됨.**
